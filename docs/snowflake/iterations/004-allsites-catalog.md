## Goal

Per spike-001's recommendation: build the catalog-with-confirmation mechanism + the first class-prefix-parameterized family canon (`*-final-cta`), and run all 7 stardust pages through one consistent mechanism (rebuilt fresh from `main` per DEC-012).

## What was built

### Phase 1 — Bridge mechanism

- **`canon/catalog.json`** — module-id → `{ canon, bemPrefix? }` mapping. Decorator fetches once per page load. Fall-through to `/canon/modules/${moduleId}.html` when unmapped (preserves iter-003 behavior for single-canon modules).
- **`blocks/stardust-module/stardust-module.js`** — rebuilt fresh per DEC-012. Adds `loadCatalog()`, `resolveCanon(moduleId)`, `applyBemPrefix(canon, prefix)`. Prefix rewrite: `__root` → `${prefix}`, `__suffix` → `${prefix}__suffix`, `--suffix` → `${prefix}--suffix`. Real utility classes (`btn`, `btn--solid-white`, `anim-enter`) unchanged because they don't start with `__` / `--` at index 0.
- **`canon/modules/final-cta.html`** — first family canon. 4 module-ids route here (6 instances: llm-final-cta, bc-final-cta ×3, aem-final-cta, aem-forrester).
- **`canon/modules/training-cta.html`** — converted to family canon via placeholders. 2 module-ids route here (training-cta, bc-training).

Local smoke test (`drafts/iter-04-smoke.html`) verified the family-canon mechanism on 6 module instances with correct prefix-rewritten BEM classes (`__title` → `aem-forrester__title`, etc.) while utility classes stayed untouched.

### Phase 4 — DA upload + publish

- **`tools/rewrite-content-urls.mjs`** — rewrites stardust-source image URLs in content files to `/media/<site>/` paths using `migrate-images.*.json` manifests as the mapping table.
- **`tools/migrate-images.sites.json`** — 16-entry manifest for AEM Sites; complements iter-04 manifests for Semrush (25 entries) and BC prototypes (9 entries).

### Phase 5 (post-deploy fix) — runtime + chrome cargo-cult

First deploy had ~50 console 404s on every page. User flagged: "rendering poorly … a lot of 404s, I consider a poor result." Root-cause investigation:

- **38 404s** were `/stardust/runtime/*.css|js` paths. Fix: commit `stardust/runtime/` (80 files / 11 MB) — iter-03 had it, iter-04 had treated it as untracked source.
- **2 404s** were `/nav.plain.html` + `/footer.plain.html`. Fix: cargo-cult `blocks/header/header.js` + `blocks/footer/footer.js` from iter-03 (custom 20-line loaders that fetch `/fragments/header.html`). The boilerplate header.js was running and fetching the wrong path.

Post-fix: console errors dropped from 61 → 1 (the remaining is `sticky-cta.js` running on pages without `.sticky-cta` — iter-03 noise inherited).

## What this iteration validates

1. **Class-prefix-parameterization mechanism works on the deployed preview.** 9 module instances across 5 pages (final-cta family on AEM Sites + LLM Optimizer + Brand Concierge + BC prototype + BC bolder; training-cta family on LLM Optimizer + Brand Concierge) render with correct prefix-rewritten BEM classes. The spike-001 strongest finding is now operationally validated, not just structurally claimed. → DEC-014.

2. **The mechanism scales to NEW pages.** AEM Sites + Semrush + BC prototypes (all not in iter-03's afbs set) render via the iter-04 mechanism. The "catalog can scale to bigger sets of URLs" claim from the spike is operationally validated for 4 new pages.

3. **Existing iter-02/iter-03 DA content is reusable** through the iter-04 mechanism without modification — module-ids are stable across iterations.

## What this iteration does NOT validate (deferred to next iterations)

1. **1:1 visual fidelity.** No pixel-diff was run on any deployed page. The user's quality goal is "all converted pages 1:1 with the originals" — that requires the pixel-diff campaign (BACKLOG) to be the closing-pass quality gate.
2. **Per-page perf (PageSpeed Insights).** Not measured.
3. **Mobile/tablet viewports.** Not exercised.
4. **Chrome variants for index vs product pages.** iter-04 ships canonical product chrome on all 7 pages; afbs index had its own gnav variant in stardust source (subbrand layout). Per iter-002's SITE-DEC-001, canonical-only with height-delta in pixel-diff is acceptable, but should be explicitly verified per page.

## Struggles + lessons (will inform iter-005 process)

User asked for explicit retrospective at iteration close. Itemized in iter-005 plan; summary:

- **Over-committed scope.** Attempted all-7-pages in one iteration; should have batched. → DEC-015.
- **`stardust/runtime/` deploy gap.** Was untracked; iter-04's first deploy was visually broken. → LEARNINGS § Deploy gotchas.
- **Chrome cargo-cult gap.** Fragments were brought forward; the custom block loaders that consume them were not. → LEARNINGS § Deploy gotchas § Chrome blocks must cargo-cult alongside fragments.
- **`aem content clone --force` was destructive.** Wiped local-only iter-04 content; required regenerating via parallel agent. → LEARNINGS § Deploy gotchas § `aem content clone --force` is destructive.
- **DA token expired silently.** First canon-upload run failed 53/53 with bare 401s. → LEARNINGS § DA token expiry causes silent 401 cascade.
- **Self-review findings deferred when fast-fixable.** The chrome issue was flagged in self-review one turn before deploy; should have been fixed before publishing.
- **Three divergent content-extractor patterns.** Programmatic / agent / cargo-cult. → BACKLOG § Consolidate content-extractor patterns.
- **`tools/node_modules/` accidentally committed.** `.gitignore` `node_modules/*` only matches root. → BACKLOG § `tools/node_modules/` cleanup.
- **Three image-manifest schemas.** No dedup, no single source of truth. → BACKLOG § Consolidate image manifests.
